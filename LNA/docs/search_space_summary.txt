Search Space Summary
====================

1. Before everything: historical broad VPP-only passive baseline.
2. Current passive database.
3. Current generated circuit space with only structural netlist constraints.

4. Active Netlist-Generator Constraints
---------------------------------------

There is no role-specific passive topology filtering in the netlist generator.
Topology constraints are expected to live in the optimization script.

Gate network:

- may come from the passive database
- may be "default"
- None is not allowed
- all-zero RLC means open

Source network:

- may come from the passive database
- may be "default"
- None means short
- all-zero RLC means open

Load network:

- may come from the passive database
- may be "default"
- None is not allowed
- all-zero RLC means open

Feedback network:

- may come from the passive database
- None means open
- all-zero RLC means open
- in the optimizer, when None feedback is allowed, the all-zero RLC feedback
  network is removed from the feedback choice axis to avoid duplicate "no
  feedback" encodings

Fixed circuit elements still present:

- gate bias resistor remains fixed
- output load resistor remains fixed
- input coupling capacitor is a fixed MIM capacitor:
  sky130_fd_pr__cap_mim_m3_2 w=10 l=10 mf=1
  This fixed input capacitor is not part of the optimization database.
- output coupling capacitor is a fixed MIM capacitor:
  sky130_fd_pr__cap_mim_m3_2 w=10 l=10 mf=1
  This fixed output capacitor is not part of the optimization database.

7. Current Make Commands
------------------------

Generate the current passive-network database:

  make generate-passive-network-multiobjective

Run NGSA-IV optimization:

  make optimize-circuits OPT_POPULATION_SIZE=24 OPT_GENERATIONS=10

Run NGSA-IV optimization with hard feasibility constraints:

  make optimize-circuits OPT_MIN_GAIN_DB=0 OPT_MAX_S11_DB=-10

If a 3 dB edge is not measured, analysis falls back to one decade around the
target frequency:

  f_3db_low = f0 / 10
  f_3db_high = f0 * 10

Run NGSA-IV optimization with four parallel ngspice evaluations per generation:

  make optimize-circuits OPT_SIM_THREADS=4

Simulate already generated circuits:

  make simulate-circuits SIM_MAX_CIRCUITS=10

Analyze existing measures without rerunning ngspice:

  make analyze-circuits SIM_MAX_CIRCUITS=10
